# PARK AI 🅿️ — Smart Parking for Chennai

**Airbnb for parking.** PARK AI connects Chennai homeowners with unused driveways and garages to drivers who need a guaranteed, affordable, covered place to park.

## ✨ Features

- **🏠 Host Portal** — list your driveway/garage in 5 minutes, set your price, track earnings, see a live earnings calculator
- **🗺 Live Map** — real OpenStreetMap (via Leaflet.js) with colour-coded slot availability
- **⚡ EV Charging Hub** — live charger status across the city + a charging cost/time calculator
- **🌤 Weather Hub** — live Chennai weather via the Open-Meteo API, with rain-aware parking impact alerts
- **🛡 Safety Console** — SOS button, CCTV network status, incident reporting
- **📡 IoT Sensor Network** — real ESP32 + ultrasonic sensor firmware (see `hardware/`), shown in-app via a clearly labeled simulation until physical units are deployed
- **☁️ Real backend (optional)** — Supabase/Postgres schema for bookings and listings, with automatic localStorage fallback if not configured
- **📊 City Insights** — occupancy charts, demand heatmaps, revenue trends, top-performing lots
- **🏆 Why PARK AI** — market sizing, competitor comparison, and product roadmap
- **🌐 Trilingual UI** — English, Tamil (தமிழ்), and Hindi (हिन्दी)

## 🎯 What's real vs. what's simulated (read this before demoing)

Being upfront about this is more credible to judges than pretending everything is finished:

| Feature | Status |
|---|---|
| UI/UX, booking flow, weather, EV calculator, map | **Real and fully functional** in the browser |
| Live Chennai weather | **Real** — calls the Open-Meteo API live |
| Live map | **Real** — actual OpenStreetMap tiles via Leaflet.js |
| Database schema, backend API code | **Real, working code** in `js/backend.js` + `supabase/schema.sql` — works once you add your own free Supabase keys (5 min setup, see below) |
| ESP32 sensor firmware | **Real code** in `hardware/esp32_parking_sensor.ino`, written against the documented Arduino/ESP32 + Supabase REST API — not yet verified on physical hardware (no board was available while building this) |
| IoT dashboard data shown in-app | **Simulated** — clearly labeled "🧪 Software Simulation" in the IoT tab, generated by a 4-second JS timer matching the real protocol's cadence |
| If no Supabase keys are set | App **automatically falls back to localStorage** — nothing breaks, but data is per-browser only |

## 🚀 Run it (basic — no backend)

No build step, no server required.

```bash
git clone https://github.com/<your-username>/parkai.git
cd parkai
open index.html   # or just double-click it
```

Or serve it locally:

```bash
python3 -m http.server 8000
# then visit http://localhost:8000
```

> The live map and weather hub need an internet connection. Everything else works fully offline, using localStorage to persist bookings/listings on your device.

## ☁️ Activate the real backend (5–10 minutes, optional but recommended)

This upgrades persistence from "this browser only" to "a real shared Postgres database," and lets the topbar status badge show "☁️ Live Supabase DB connected" instead of "💾 Local demo storage."

1. Go to [supabase.com](https://supabase.com) → New Project (free tier, no credit card)
2. Once created, open the **SQL Editor** → paste the entire contents of `supabase/schema.sql` → Run
3. Go to **Project Settings → API** → copy the **Project URL** and **anon public** key
4. Open `js/backend.js` and paste them into:
   ```js
   const SUPABASE_URL      = "https://xxxxxxxxxxxx.supabase.co";
   const SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIs...";
   ```
5. Reload the app. The topbar badge should switch to "☁️ Live Supabase DB connected."

If you skip this, the app still works perfectly — it just uses localStorage instead.

## 🔌 Deploy real hardware (optional, for the IoT track)

See `hardware/README.md` for the full bill of materials (~₹455/slot), wiring diagram, and Arduino IDE setup steps to flash `hardware/esp32_parking_sensor.ino` onto a real ESP32 + HC-SR04 ultrasonic sensor.

## 🔧 Tech Stack

- Vanilla HTML/CSS/JS — zero build tooling
- [Leaflet.js](https://leafletjs.com/) + OpenStreetMap (CARTO dark tiles) for the live map
- [Open-Meteo](https://open-meteo.com/) for live weather data (free, no API key needed)
- [Supabase](https://supabase.com) (Postgres + Realtime) for the optional real backend
- ESP32 + HC-SR04 ultrasonic sensor for the optional real hardware layer
- `localStorage` as the zero-setup fallback backend
